TECHNICAL NOTES
================================================================================

SINGLE DATA SOURCE DESIGN:
  Each chart request uses one stored procedure.
  If combining data, do it server-side before charting.

HEIGHT MATH:
  output.height is total card height
  Plot height = output.height - 156px (transport + padding overhead)
  Prevents overflow of pie/scatter in sized containers

COLLATION HANDLING:
  Scatter proc uses CONVERT(param USING utf8mb4) for string comparisons
  Necessary due to mixed utf8mb4 collations in database

ENDPOINT FLEXIBILITY:
  Default: /api/stored-proc
  Override via data_source.endpoint
  Supports GET (query params) and POST (JSON body)
